```yaml
Agent 认知模型 (Cognitive Architecture):

感知层 (Perception):
  输入模态:
    - 结构化数据: 命令输出、系统指标、JSON日志
    - 非结构化文本: 错误堆栈、README文档、用户对话
    - 时序信号: 资源使用曲线、网络延迟波动

    - 上下文关联: 将当前输入与历史状态关联

认知层 (Cognition):
  工作记忆 (Working Memory): 当前任务上下文，限制7±2个信息块
  长期记忆 (Long-term Memory): 通过RAG访问知识库
  推理引擎:
    - 演绎推理: 基于已知规则推导（如"Ubuntu 22.04没有python2"）
    - 归纳推理: 从案例总结模式（如"Node项目常需build-essential"）

执行层 (Action):
  工具使用 (Tool Use):
    - 命令生成器: 生成shell/python/sql命令
    - API调用器: 调用内部服务或外部API

  行动选择策略:
    - 贪婪策略: 高置信度时直接执行
    - 保守策略: 涉及数据修改时需确认
    - 探索策略: 遇到未知错误时尝试多种修复

元认知层 (Meta-cognition):
  自我监控: 评估自身置信度，低置信度时请求协助
  学习机制: 从执行结果更新内部模型
  解释生成: 生成自然语言解释说明决策理由
```

**实时输出处理**：

```python
class ExecutionStreamer:
    """
    实时处理命令输出，提取关键信息并流式返回给前端
    """
    
    def process_stream(self, ssh_channel):
        buffer = ""
        progress_pattern = re.compile(r'(\d+)%')
        error_keywords = ['error', 'failed', 'fatal', 'exception', 'cannot']
        
        for line in iter(ssh_channel.readline, ''):
            # 1. 实时推送到前端（WebSocket）
            self.websocket.send(json.dumps({
                "type": "output",
                "timestamp": time.time(),
                "content": line,
                "step_id": self.current_step_id
            }))
            
            # 2. 进度提取（如npm install的进度条）
            if match := progress_pattern.search(line):
                progress = int(match.group(1))
                self.update_progress(progress)
            
            # 3. 错误预警（提前发现潜在失败）
            if any(keyword in line.lower() for keyword in error_keywords):
                self.flag_potential_error(line)
                
            # 4. 上下文积累（用于后续诊断）
            buffer += line
            if len(buffer) > 10000:  # 限制上下文大小
                buffer = buffer[-5000:]
        
        return buffer
```

**多轮对话排错协议**：

```yaml

初始状态:
  context: 完整执行日志（最后1000行）
  attempts: 0
  max_attempts: 5
  
对话循环:
  while attempts < max_attempts:
    # 1. 诊断
    diagnosis = troubleshooter.analyze(context)
    
    if diagnosis.confidence > 0.9:
        # 高置信度自动修复
        execute_fix(diagnosis.solution)
        result = verify_fix()
        
        if result.success:
            record_success(diagnosis)  # 记录到知识库
            break
        else:
            context += result.new_logs
            attempts += 1
            continue  # 进入下一轮
            
    elif diagnosis.confidence > 0.6:
        # 中置信度建议用户确认
        user_choice = present_options([
            "自动应用建议修复",
            "查看详细分析",
            "跳过此步骤",
            "请求人工支持"
        ])
        
        if user_choice == "自动应用":
            execute_fix(diagnosis.solution)
        elif user_choice == "跳过":
            mark_step_as_skipped()
            break
        else:
            escalate_to_human()
            break
            
    else:
        # 低置信度立即升级
        escalate_to_human(
            reason="无法确定根因",
            context=context,
            attempts_made=attempts
        )
        break

升级后处理:
  human_resolution = wait_for_human_input()
  learn_from_human(human_resolution)  # 强化学习更新

## 第二部分：RAG 知识库系统工程

### 2.1 知识库数据架构

#### 2.1.1 多层知识表示

```
知识库层级结构:

L1: 原始日志层 (Raw Logs)
    └── 完整的命令输出、错误堆栈、系统日志
    └── 存储: 对象存储 (S3/MinIO)，压缩保留90天
    
L2: 事件层 (Events)
    └── 结构化执行事件
    └── Schema: {timestamp, step, action, result, duration, output_hash}
    └── 存储: ClickHouse (时序数据库)
    
L3: 案例层 (Cases)
    └── 完整的部署案例（成功/失败）
    └── 包含: 环境指纹、执行路径、错误记录、解决方案
    └── 存储: PostgreSQL + 向量索引
    
L4: 模式层 (Patterns)

```python
class ContextOptimizer:
    """
    优化RAG上下文，适配LLM的token限制
    """
    
    def optimize_context(self, retrieved_cases, max_tokens=4000):
        """
        智能选择和组织检索结果
        """
        context_parts = []
        current_tokens = 0
        
        # 优先级1: 完全匹配的案例（相同OS+技术栈）
        exact_matches = [c for c in retrieved_cases if self.is_exact_match(c)]
        for case in exact_matches[:2]:  # 最多2个
            text = self.format_case(case, detail_level="full")
            tokens = self.estimate_tokens(text)
            if current_tokens + tokens < max_tokens * 0.6:
                context_parts.append(text)
                current_tokens += tokens
        
        # 优先级2: 相似技术栈但不同OS（用于适配参考）
        similar_cases = [c for c in retrieved_cases if not self.is_exact_match(c)]
        for case in similar_cases:
            text = self.format_case(case, detail_level="summary")
            tokens = self.estimate_tokens(text)
            if current_tokens + tokens < max_tokens * 0.3:
                context_parts.append(text)
                current_tokens += tokens
        
        # 优先级3: 通用解决方案（作为fallback）
        general_solution = self.get_general_solution()
        text = self.format_case(general_solution, detail_level="brief")
        tokens = self.estimate_tokens(text)
        if current_tokens + tokens < max_tokens * 0.1:
            context_parts.append(text)
        
        return "\n\n---\n\n".join(context_parts)
    
    def format_case(self, case, detail_level):
        """
        根据重要性调整细节级别
        """
        if detail_level == "full":
            return f"""
            Case ID: {case.id}
            Environment: {case.os} {case.os_version}, {case.tech_stack}
            Problem: {case.error_description}
            Root Cause: {case.root_cause}
            Solution: {case.solution_commands}
            Verification: {case.verification_command}
            Success Rate: {case.success_rate}
            """
        
        elif detail_level == "summary":
            return f"""
            Reference Case ({case.os}):
            Issue: {case.error_type}
            Fix: {case.solution_summary}
            Note: OS differs from target, may need adaptation
            """
        
        else:  # brief
            return f"General approach: {case.general_strategy}"
```

```python
class FailureAnalyzer:
    """
    从失败中提取知识，构建"坑点地图"
    """
    
    def analyze_failure(self, deployment_record):
        """
        深度分析失败案例
        """
        insights = {}
        
        # 1. 失败模式识别
        failure_signature = self.generate_signature(deployment_record)
        similar_failures = self.find_similar_failures(failure_signature)
        
        if len(similar_failures) > 5:
            insights['systemic_issue'] = {
                'description': '检测到系统性问题',
                'frequency': len(similar_failures),
                'affected_stacks': self.extract_common_factors(similar_failures),
                'recommendation': '需要更新基础镜像或文档'
            }
        
        # 2. 恢复路径分析
        if deployment_record.recovery_attempts:
            successful_recovery = [
                r for r in deployment_record.recovery_attempts 
                if r.result == 'success'
            ]
            if successful_recovery:
                best_recovery = max(
                    successful_recovery, 
                    key=lambda x: x.efficiency_score
                )
                insights['recovery_pattern'] = {
                    'trigger_condition': best_recovery.trigger_error,
                    'effective_action': best_recovery.action,
                    'time_to_recover': best_recovery.duration
                }
        
        # 3. 预防性知识提取
        early_warnings = self.detect_early_warnings(deployment_record.logs)
        if early_warnings:
            insights['predictive_indicators'] = early_warnings
        
        return insights
    
    def generate_prevention_rules(self, failure_insights):
        """
        将失败洞察转化为预防规则
        """
        rules = []
        
        for insight in failure_insights:
            if 'predictive_indicators' in insight:
                rule = {
                    'if': {
                        'pattern': insight['predictive_indicators'],
                        'confidence': insight['confidence']
                    },
                    'then': {
                        'action': 'preemptive_fix',
                        'target': insight['potential_failure_point']
                    },
                    'priority': 'high' if insight['severity'] == 'critical' else 'normal'
                }
                rules.append(rule)
        
        return rules
```

### 3.2 状态一致性管理

```python
class DeploymentStateManager:
    """
    部署状态机管理，确保强一致性
    """
    
    STATES = [
        'PENDING',           # 等待开始
        'ENV_PREPARING',     # 准备运行环境
        'CODE_FETCHING',     # 拉取代码
        'DEPENDENCY_INSTALLING', # 安装依赖
        'BUILDING',          # 编译构建
        'CONFIGURING',       # 配置服务
        'DEPLOYING',         # 实际部署
        'VERIFYING',         # 健康检查
        'COMPLETED',         # 成功完成
        'ROLLING_BACK',      # 回滚中
        'FAILED',            # 失败
        'PAUSED'             # 暂停（等待人工）
    ]
    
    TRANSITIONS = {
        'PENDING': ['ENV_PREPARING', 'FAILED'],
        'ENV_PREPARING': ['CODE_FETCHING', 'ROLLING_BACK', 'FAILED'],
        'CODE_FETCHING': ['DEPENDENCY_INSTALLING', 'ROLLING_BACK', 'FAILED'],
        'DEPENDENCY_INSTALLING': ['BUILDING', 'ROLLING_BACK', 'FAILED', 'PAUSED'],
        'BUILDING': ['CONFIGURING', 'ROLLING_BACK', 'FAILED', 'PAUSED'],
        'CONFIGURING': ['DEPLOYING', 'ROLLING_BACK', 'FAILED'],
        'DEPLOYING': ['VERIFYING', 'ROLLING_BACK', 'FAILED'],
        'VERIFYING': ['COMPLETED', 'ROLLING_BACK', 'FAILED'],
        'ROLLING_BACK': ['FAILED', 'PENDING'],  # 可重试
        'PAUSED': ['BUILDING', 'ROLLING_BACK', 'FAILED'],  # 继续或放弃
    }
    
    def transition(self, deployment_id, new_state, context):
        """
        状态转换，使用分布式锁保证一致性
        """
        with self.redis.lock(f"lock:deployment:{deployment_id}", timeout=30):
            current = self.get_state(deployment_id)
            
            if new_state not in self.TRANSITIONS.get(current, []):
                raise InvalidStateTransition(
                    f"Cannot transition from {current} to {new_state}"
                )
            
            # 记录状态变更历史
            self.audit_log.append({
                'deployment_id': deployment_id,
                'from': current,
                'to': new_state,
                'timestamp': datetime.utcnow(),
                'context': context,
                'actor': context.get('agent_id') or context.get('user_id')
            })
            
            # 更新状态
            self.redis.hset(
                f"deployment:{deployment_id}",
                mapping={
                    'state': new_state,
                    'updated_at': datetime.utcnow().isoformat(),
                    'context': json.dumps(context)
                }
            )
            
            # 发布状态变更事件
            self.event_bus.publish('deployment.state_changed', {
                'deployment_id': deployment_id,
                'new_state': new_state,
                'previous_state': current
            })
            
            return True
```

### 4.2 命令安全沙箱

```python
class CommandSandbox:
    """
    AI生成命令的安全检查与沙箱执行
    """
    
    DANGEROUS_PATTERNS = [
        # 破坏性操作
        (r'\brm\s+-rf\s+/', 'CRITICAL', 'Attempting to delete root filesystem'),
        (r'\bmkfs\.', 'CRITICAL', 'Filesystem formatting detected'),
        (r'\bdd\s+if=.*\bof=/dev/[sh]d', 'CRITICAL', 'Direct disk writing'),
        
        # 权限提升
        (r'\bchmod\s+.*777\b', 'HIGH', 'Overly permissive permissions'),
        (r'\bchown\s+-R\s+root\b', 'HIGH', 'Changing ownership to root'),
        
        # 网络风险
        (r'\bwget\s+.*\|.*\bsh\b', 'HIGH', 'Pipe to shell anti-pattern'),
        (r'\bcurl\s+.*\|.*\bsh\b', 'HIGH', 'Pipe to shell anti-pattern'),
        
        # 信息泄露
        (r'\bcat\s+/etc/shadow', 'MEDIUM', 'Reading sensitive files'),
        (r'\benv\s+.*\b(API_KEY|PRIVATE|SECRET)', 'MEDIUM', 'Potential secret exposure'),
    ]
    
    ALLOWED_COMMANDS = {
        'apt-get': {'install', 'update', 'upgrade', 'remove', 'purge'},
        'yum': {'install', 'update', 'remove', 'search'},
        'systemctl': {'start', 'stop', 'restart', 'status', 'enable', 'disable'},
        'docker': {'pull', 'run', 'stop', 'rm', 'ps', 'logs', 'build'},
        'git': {'clone', 'pull', 'checkout', 'fetch', 'reset'},
        'npm': {'install', 'run', 'build', 'start', 'test'},
        'pip': {'install', 'uninstall', 'freeze', 'list'},
    }
    
    def validate(self, command: str, context: dict) -> ValidationResult:
        """
        多层级验证
        """
        # 层级1: 正则模式匹配
        for pattern, severity, reason in self.DANGEROUS_PATTERNS:
            if re.search(pattern, command, re.IGNORECASE):
                return ValidationResult(
                    allowed=False,
                    severity=severity,
                    reason=reason,
                    action='BLOCK'
                )
        
        # 层级2: 命令白名单检查
        cmd_parts = shlex.split(command)
        base_cmd = cmd_parts[0]
        
        if base_cmd not in self.ALLOWED_COMMANDS:
            return ValidationResult(
                allowed=False,
                severity='MEDIUM',
                reason=f'Command {base_cmd} not in whitelist',
                action='REVIEW_REQUIRED'
            )
        
        # 检查子命令
        if len(cmd_parts) > 1:
            sub_cmd = cmd_parts[1].lstrip('-')
            allowed_subs = self.ALLOWED_COMMANDS[base_cmd]
            if sub_cmd not in allowed_subs:
                return ValidationResult(
                    allowed=False,
                    severity='LOW',
                    reason=f'Subcommand {sub_cmd} not allowed for {base_cmd}',
                    action='REVIEW_REQUIRED'
                )
        
        # 层级3: 上下文感知检查
        if 'production' in context.get('environment', ''):
            if base_cmd in ['apt-get', 'yum'] and 'upgrade' in command:
                return ValidationResult(
                    allowed=False,
                    severity='HIGH',
                    reason='System upgrade in production requires approval',
                    action='REQUIRE_APPROVAL'
                )
        
        # 层级4: 语义分析（使用LLM）
        semantic_risk = self.llm_analyze(command, context)
        if semantic_risk.score > 0.8:
            return ValidationResult(
                allowed=False,
                severity='HIGH',
                reason=f'Semantic analysis detected risk: {semantic_risk.explanation}',
                action='REQUIRE_APPROVAL'
            )
        
        return ValidationResult(allowed=True)
    
    def execute_in_sandbox(self, command: str, server_connection):
        """
        在受限环境中执行命令
        """
        # 使用nsenter进入受限命名空间
        sandbox_cmd = f"""
        unshare --fork --pid --mount-proc --net \
        --user --map-root-user \
        chroot /var/sandbox/servermind \
        /bin/bash -c {shlex.quote(command)}
        """
        
        # 资源限制
        limits = {
            'cpu_quota': 100000,  # 100ms per 100ms
            'memory_limit': '512m',
            'timeout': 300,
            'max_file_size': '100m',
            'max_processes': 50
        }
        
        return server_connection.execute(sandbox_cmd, limits=limits)
```
